# GigaWebZone — Corporate Website

The GigaWebZone LLP marketing site — a full-stack **Next.js (App Router) + TypeScript**
build, repositioning the brand from a WordPress shop to a full-stack digital build studio.

Light-default with a dark-mode toggle, a token-based design system, SEO + schema, an
accessible component library, and a CRM-ready enquiry form.

## Tech

- **Next.js 15** (App Router, SSR/SSG) · **TypeScript**
- **next-themes** (light default, dark toggle, `data-theme`, no-flash SSR)
- **next/font** — Space Grotesk · Manrope · Space Mono
- Token-based CSS ported from the design system (`src/styles/`)
- `output: "standalone"` for lean Node deploys

## Local development

```bash
npm install
npm run dev      # http://localhost:3000
npm run build    # production build (also runs lint + type-check)
npm run start    # serve the production build
```

Copy `.env.example` → `.env.local` and fill values as needed (all optional today).

## Project structure

```
src/

  components/     # Header, Footer, Hero, service/, industries/, portfolio/, contact/, legal/, ui/
  lib/            # data/ (projects, industries, case studies), hooks, site, seo
  styles/         # gwz.css + gwz-pages.css (ported), extras.css (additions)
public/assets/    # logos + founder image
design-reference/ # the companion visual spec (HTML/CSS/screens)
```

## SEO

- Per-page `metadata` (title/description/canonical/OpenGraph/Twitter).
- JSON-LD: Organization + WebSite (site-wide), BreadcrumbList, Service, AboutPage,
  ContactPage and CreativeWork (case studies).
- `sitemap.xml` and `robots.txt` are generated by `src/app/sitemap.ts` and
  `src/app/robots.ts`.
- **Submit this sitemap in Google Search Console:**
  `https://www.gigawebzone.com/sitemap.xml`

> This is a fresh launch — there is no legacy WordPress URL/redirect map to maintain.
> Schema uses an **Organization** (not LocalBusiness), so the site is not optimised for
> local-only ranking; the business serves clients nationally and internationally.

## Analytics & consent

GA4 is **off by default**. Set `NEXT_PUBLIC_GA_ID` to enable it; a DPDP-friendly cookie
consent banner then appears, and GA4 loads only after the visitor accepts.

## Deploying to Hostinger (Node.js app)

This is a Node server app (not a static export), so server actions / API routes / ISR
keep working.

1. **Create a Node.js application** in hPanel (Node 18+), pointing at this repo.
2. **Install + build**
   ```bash
   npm install
   npm run build
   ```
3. **Start command:** `npm run start`
   - Next listens on `process.env.PORT` (Hostinger sets this automatically).
4. **Environment variables** (hPanel → Node app → Environment): add `NEXT_PUBLIC_GA_ID`
   etc. as needed, then rebuild.
5. **HTTPS:** enable SSL for the domain. Security headers (HSTS, X-Frame-Options,
   Referrer-Policy, Permissions-Policy, nosniff) are set in `next.config.mjs`.

### Leaner alternative — standalone server

`output: "standalone"` emits a self-contained server. After `npm run build`:

```bash
cp -r public .next/standalone/public
cp -r .next/static .next/standalone/.next/static
node .next/standalone/server.js     # honors PORT
```

Use whichever your Hostinger plan supports; `npm run start` is the simplest.
